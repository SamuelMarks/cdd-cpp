# Publish Output

This document focuses on the generated SDK and Server code generated by the CLI.

## Automated Updates

To keep your generated SDK packages synchronized with the server's OpenAPI spec, utilize a cron job inside a GitHub Action or GitLab Pipeline.

```yml
name: Sync SDK

on:
  schedule:
    - cron: '0 0 * * *'

jobs:
  sync_sdk:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - name: Setup CDD CLI
        run: sudo apt install cdd-cpp || make install
      - name: Sync
        run: cdd-cpp from_openapi to_sdk -i remote_spec.json -o ./sdk/
      - name: Commit
        run: |
          git add ./sdk/
          git commit -m "Auto-sync SDK" || true
          git push
```

## Distribution

The resulting Client SDK CLI, Server Routes, or Models can be natively distributed using their appropriate package managers (e.g. `pip`, `npm`, `cargo`, `vcpkg`).
